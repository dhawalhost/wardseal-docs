# Environment Configuration

WardSeal microservices are configured using environment variables. This guide provides a detailed reference for all core configuration options.

## Global Variables

These variables are common across most services.

| Variable | Description | Default |
| :--- | :--- | :--- |
| `DB_HOST` | Hostname of the PostgreSQL database. | `localhost` |
| `DB_PORT` | Port of the PostgreSQL database. | `5432` |
| `DB_USER` | Database username. | `user` |
| `DB_PASSWORD` | Database password. | `password` |
| `DB_NAME` | Name of the WardSeal database. | `identity_platform` |
| `DB_SSLMODE` | PostgreSQL SSL mode (`disable`, `require`). | `disable` |
| `REDIS_ADDR` | Address of the Redis cache (e.g., `localhost:6379`). | `localhost:6379` |
| `LOG_LEVEL` | Logging verbosity (`debug`, `info`, `warn`, `error`). | `info` |

## Auth Service (`authsvc`)

| Variable | Description |
| :--- | :--- |
| `AUTH_SERVICE_URL` | The public URL of the auth service (e.g., `http://auth.wardseal.local`). |
| `DIRECTORY_SERVICE_URL` | The internal URL used to communicate with `dirsvc`. |
| `CORS_ALLOWED_ORIGINS` | Comma-separated list of origins allowed to make requests. |
| `JWT_PRIVATE_KEY_PATH` | Path to the private key used for signing tokens. |
| `JWT_PUBLIC_KEY_PATH` | Path to the public key used for verifying tokens. |
| `KMS_PROVIDER` | KMS backend to use (`local`, `vault`, `aws`, `gcp`). |
| `DEPLOYMENT_MODE` | Deployment type (`saas` or `onprem`). |

## Directory Service (`dirsvc`)

| Variable | Description |
| :--- | :--- |
| `SCIM_ENABLED` | Whether to enable SCIM 2.0 endpoints. |
| `TENANT_ISOLATION_STRICT` | If true, fails requests without a valid `tenant_id`. |

## Governance Service (`govsvc`)

| Variable | Description |
| :--- | :--- |
| `ADMIN_INTERNAL_API_KEY` | Secret key used for internal service-to-service auth. |
| `WEBHOOK_RETRY_LIMIT` | Number of times to retry failed audit webhooks. |

## Provisioning Service (`provsvc`)

| Variable | Description |
| :--- | :--- |
| `PROVISIONING_SYNC_INTERVAL` | Interval between background sync runs (e.g., `5m`). |
| `MAX_CONCURRENT_PROVISIONERS` | Limit of parallel SCIM sync operations. |

## UI Configuration (Build Arguments)

For the frontend UIs, these variables must be provided during the **docker build** process as `--build-arg`.

### Admin Console (`adminui`)
- `VITE_AUTHSVC_URL`: Public URL of the auth service.
- `VITE_DIRSVC_URL`: Public URL of the directory API.
- `VITE_GOVSVC_URL`: Public URL of the governance API.

### Landing Page (`landingui`)
- `VITE_SITE_BASE_URL`: Base URL of the marketing site.
- `VITE_CONSOLE_BASE_URL`: URL of the admin console.
- `VITE_SUPPORT_EMAIL`: Contact email for support links.
