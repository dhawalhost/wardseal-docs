# Groups

Groups allow you to organize users and manage permissions at scale.

## Overview

Instead of assigning applications and permissions to individual users, you should assign them to groups. Users then inherit the permissions of all groups they belong to.

## Group Types

### Static Groups
Manually add or remove users from these groups using the Admin Console or API.

### Dynamic Groups (Coming Soon)
Define rules based on user attributes (e.g., `department == "Engineering"`) to automatically populate group membership.

## Assigning Applications

Assign applications to a group to grant access to all its members. This is the recommended way to manage application access in WardSeal.
